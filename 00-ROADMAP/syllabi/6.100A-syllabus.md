## Descripción del Curso

Introducción a la programación en Python con énfasis en pensar computacionalmente. No se trata de memorizar sintaxis — se trata de aprender a descomponer problemas complejos en pasos simples y traducirlos a código. Al finalizar este curso, el estudiante puede construir programas completos desde cero sin asistencia, leer código ajeno y entenderlo, y tiene los fundamentos para abordar cualquier curso del programa.

Este curso es el equivalente directo de 6.100A tal como se enseña en MIT, comprimido en formato intensivo de 4 semanas.

---

## Objetivos de Aprendizaje

Al completar 6.100A, el estudiante será capaz de:

1. Escribir programas Python completos que resuelven problemas reales
2. Usar todos los tipos de datos fundamentales y saber cuándo aplicar cada uno
3. Diseñar funciones bien estructuradas con responsabilidad única
4. Aplicar estructuras de control para modelar cualquier lógica de negocio
5. Usar clases y objetos para organizar código complejo
6. Manejar errores de manera robusta con try/except
7. Escribir tests básicos para verificar que el código funciona
8. Leer documentación oficial y resolver problemas sin tutorial

### SEMANA 2 — Control Flow & Functions
**"El código toma decisiones y aprende a repetirse"**

#### Temas
| Día | Tema | Conceptos clave |
|-----|------|-----------------|
| Lunes | Branching | if · elif · else · condiciones anidadas · operador ternario |
| Martes | Loops I | while · break · continue · loops infinitos controlados |
| Miércoles | Loops II | for · range() · iteración sobre strings y listas · enumerate() |
| Jueves | Funciones | def · return · parámetros · scope · docstrings · funciones como valores |
| Viernes | Funciones avanzadas + Problem Set 2 | *args · **kwargs · funciones que retornan funciones · recursión básica |

#### Conceptos críticos de la semana
- **Scope:** variables locales vs globales — una función tiene su propio espacio de nombres
- **Return vs Print:** return es para computación, print es para visualización. No son lo mismo.
- **Abstracción funcional:** una función debería hacer UNA cosa bien. No diez cosas mal.
- **DRY principle:** Don't Repeat Yourself — si copias y pegas código, probablemente necesitas una función

### SEMANA 3 — Data Structures & Object-Oriented Programming
**"Organizando datos y diseñando sistemas"**

#### Temas
| Día | Tema | Conceptos clave |
|-----|------|-----------------|
| Lunes | Listas | Creación · indexing · slicing · append · remove · sort · comprensiones |
| Martes | Diccionarios y Tuplas | dict: keys/values/items · tuplas inmutables · cuándo usar cada uno |
| Miércoles | Conjuntos y módulos | set · operaciones de conjuntos · import · módulos estándar (os, sys, math, random, datetime) |
| Jueves | OOP I | Clases · instancias · `__init__` · métodos · atributos · `self` |
| Viernes | OOP II + Problem Set 3 | Herencia · `super()` · métodos especiales (`__str__`, `__len__`, `__eq__`) |

#### Conceptos críticos de la semana
- **Mutabilidad:** listas y dicts son mutables · strings y tuplas son inmutables · esto importa cuando pasas datos a funciones
- **Referencia vs copia:** `b = a` no copia una lista, copia la referencia · usa `b = a.copy()` para copiar
- **Encapsulamiento:** una clase agrupa datos (atributos) y comportamiento (métodos) que pertenecen juntos
- **`__init__` no es el constructor:** es el inicializador. Python ya creó el objeto cuando entra a `__init__`.

```
PS3.3 — Analizador de texto avanzado
  Recibe un archivo .txt como argumento

  Usa collections.Counter
  Imprime reporte formateado
```

### SEMANA 4 — Error Handling, Testing & Professional Python
**"Escribir código que funciona en el mundo real"**

#### Temas
| Día | Tema | Conceptos clave |
|-----|------|-----------------|
| Lunes | Manejo de errores | try · except · else · finally · raise · excepciones personalizadas |
| Martes | Archivos e I/O | open() · read/write · pathlib · context managers (with) · JSON · CSV |
| Miércoles | Testing con pytest | Unit tests · fixtures · assert · test-driven development intro |
| Jueves | Python profesional | Virtual environments · pip · requirements.txt · PEP 8 · ruff/flake8 · type hints básicos |
| Viernes | Final Exam + Mini-proyecto 3 | Problema abierto sin guía |

#### Conceptos críticos de la semana
- **Fail loudly:** un error explícito es mejor que datos corruptos silenciosos · raise siempre que algo no pueda continuar
- **Context managers:** `with open(...)` garantiza que el archivo se cierra aunque el código falle
- **Tests son documentación:** un buen test describe qué debe hacer la función, no solo que "funciona"
- **Virtual environments:** cada proyecto tiene sus propias dependencias · nunca instalar en el sistema global
